Links

Links within paragraphs

By default, links contained within a paragraph <p> tag will be teal .text_teal in color.

  • Hovering over the link with the mouse will appear underlined and change the cursor to a hand
  • Visited links will appear the default browser visited link color and still have an underline.
  • Links not contained within a paragraph should follow the individual component/pattern styling as defined
    • In general, links contained within interface components will not be underlined

Alerts & Notifications

Alerts

Purpose

Users need to be aware of important information related to their current activity.

Description

  • Contextual alerts are positioned above the area or form they relate to
  • They are often triggered by a user interaction
  • They can be closable and can have other action buttons or links.
  • Alerts with low importance (info or success) can close automatically after 5 seconds (if desired)
  • Alerts with high importance (warning or danger) should not close automatically, unless the situation has been resolved in some other way

Types of Alerts

  • Info alerts should be used when there are tips or information that a user can benefit from.
  • Success alerts should be used when an action was performed successfully.
  • Warning alerts should be used when an action is out of the ordinary or might not be desired.
  • Danger alerts should be used when the system has failed to perform an action, or when the user has made an error.

Buttons

Button styles

There are 3 button styles, you can see them here

  • Orange buttons are used as the primary action buttons
  • Grey buttons are used as the secondary action buttons. When a button is disabled, it also appears grey.
  • Search buttons are slightly darker grey than the secondary action buttons.
  • Internal buttons are blue colored to distinguish them from customer facing buttons and only appear on internal pages.
  • All buttons to look like this:
    • .btnb big oj
    • .btnb medium oj
    • .btnb medium_110 oj
    • .btnb sml oj
  • Buttons could have a slight shadow on the bottom, using Bootstrap with Google Material Design to make it obvious that there is a button (not all users know that a flat color is actually a clickable button - check this example ).
  • Our buttons have a 3px rounded corner / border-radius.
  • If something is not a button, it should not look like a button. There should be no rounded corners either.

Interactive elements

Any element that is interactive, needs to have the following states:

  • hover
  • active
  • disabled
  • mouse-down / focus
  • proper cursor change (plus, hand)
